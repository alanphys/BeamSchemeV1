File Import ErrorsÂ¶

If BeamScheme does not recognise a file or encounters a problem during the import of the file an error will be thrown. BeamScheme firstly examines the file extension to determine the type of file and then the file header. If BeamScheme can not read the file or find the data in the body of the file that it is looking for it will throw one of the following errors.

  • Text file not recognised!
  • Not a recognised MapCheck text file!
  • Not a recognised IBA opg file!
  • Not a recognised PTW mcc file!
  • Not a recognised XIO text file!
  • Not a recognised BrainLab iPlan file!
  • Could not read RAW text file!
  • Not a recognised HIS file!
  • Not a recognised Windows bitmap file!
  • This software can not read compressed or 24-bit color files
  • Unable to load DICOM header segment. Is this really a DICOM compliant file?
  • This file does not have enough data for the image size
  • DICOM file error, no data found!
  • DICOM file error, corrupt file
